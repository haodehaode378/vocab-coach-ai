# AI Agent 记单词项目｜04 ADR

## ADR-001 先做单 Agent 再演进多 Agent
- 决策：MVP 采用单 Agent + Tool 调用。
- 原因：复杂度可控，利于快速验证。

## ADR-002 复习算法采用 SM-2
- 决策：先用 SM-2 作为默认 SRS。
- 原因：成熟稳定，落地成本低。

## ADR-003 数据库先 SQLite，预留 Postgres 迁移
- 决策：开发/内测阶段使用 SQLite。
- 原因：部署简单、迭代快。

## ADR-004 AI 内容生成必须支持降级
- 决策：所有 AI 生成接口增加本地模板回退。
- 原因：避免 AI 服务抖动影响主流程。
